### 体重页面空状态优化 ###
优化体重页面在无数据时的展示效果，提供友好的空状态引导，提升用户首次使用体验

# 体重页面空状态优化执行计划

## 一、设计方案

### 1.1 空状态展示策略
当 `records.length === 0` 时：
- **隐藏组件**：WeightChart、WeightStatsRow（无意义的空数据展示）
- **保留组件**：WeightStatusCard（显示"——"，可设置目标）、WeightFAB（引导录入）
- **新增组件**：WeightEmptyState（友好的空状态引导）

### 1.2 空状态组件设计
参考 `components/home/EmptyState.tsx` 的设计风格：
- **插图**：📊 或 ⚖️ emoji
- **标题**：开始记录体重
- **描述**：记录每一次变化，见证每一次进步
- **引导按钮**：点击右下角 ➕ 开始记录（带动画指向 FAB）

### 1.3 动画效果
- 空状态组件：FadeInDown 入场动画
- FAB 按钮：在空状态时添加脉冲动画，吸引用户注意

## 二、实现步骤

### 2.1 创建空状态组件
**文件**：`components/weight/WeightEmptyState.tsx`

**功能**：
- 显示友好的空状态插图和文案
- 引导用户点击 FAB 按钮录入数据
- 使用 FadeInDown 动画

### 2.2 修改主页面逻辑
**文件**：`app/(tabs)/weight.tsx`

**修改点**：
- 添加 `isEmpty` 判断：`records.length === 0`
- 条件渲染：
  - `isEmpty === true`：显示 WeightStatusCard + WeightEmptyState
  - `isEmpty === false`：显示完整内容（现有逻辑）

### 2.3 优化 WeightStatusCard
**文件**：`components/weight/WeightStatusCard.tsx`

**修改点**：
- 空数据时隐藏"较上次变化"区域
- 空数据时不显示 BMI 卡片
- 保留目标卡片（引导用户设置目标）

### 2.4 优化 FAB 按钮（可选）
**文件**：`components/weight/WeightFAB.tsx`

**可选优化**：
- 添加 `isEmpty` prop
- 空状态时添加脉冲动画（withRepeat + withSequence）

## 三、文件清单

### 新增文件
- `components/weight/WeightEmptyState.tsx`

### 修改文件
- `app/(tabs)/weight.tsx`
- `components/weight/WeightStatusCard.tsx`（可选，已有部分空状态处理）
- `components/weight/WeightFAB.tsx`（可选，添加脉冲动画）

## 四、验证要点

1. 删除所有体重记录后，页面显示空状态
2. 空状态下点击 FAB 可正常录入数据
3. 录入第一条数据后，空状态消失，显示完整内容
4. 动画流畅，无闪烁
5. 空状态文案友好，引导清晰
